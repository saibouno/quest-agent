# Quest Agent v0.1 PRD

## 一文定義
Quest Agent は、目標はあるのに進め方で止まりがちな人のために、目標を実行可能な流れに変え、実現確率を高める伴走エージェントである。

## 対象ユーザー
- 個人で事業、サービス、作品、プロジェクトを本気で前に進めたい人
- 目標はあるが、分解、着手、継続、立て直しで止まりやすい人
- やる気不足ではなく、`目標 -> 実行可能な流れ` への変換に詰まっている人

## コア価値
- 明確さ: 達成条件と次の一手を明らかにする
- 着手率: 始めやすいサイズまで落とす
- 継続率: 毎日少しでも前進できる状態を作る
- 再始動率: 止まったあとに立て直せるようにする

## 非対象
- 単なるタスク列挙アプリ
- モチベーションだけを扱うサービス
- 最初から人生全体を管理する OS
- 高度なマルチエージェント前提の自律実行システム

## v0.1 で提供する体験
1. Goal Intake で目標、Why、期限、成功条件、制約、現在地、不安を整理する
2. Quest Map で goal を milestone と quest に分解する
3. Today’s Quests で今日やる 1〜3 件を提示する
4. Blocker を記録し、次の一手や代替ルートを返す
5. Weekly Review で振り返り、設計を reroute する

## ユーザーに求める入力
- 何を実現したいか
- なぜやりたいか
- いつまでにやるか
- 何が不安か
- 今日はどこまでできそうか

## 主要ユースケース
### 1. 目標を整理したい
ユーザーは曖昧な目標を書き出し、AI が goal draft と success criteria を叩き台として返す。

### 2. 道筋を仮で作りたい
Goal から milestone と quest 候補を生成し、順番と優先度を持つ route を得る。

### 3. 今日前進したい
現在の quest と blocker を踏まえ、今日やるべき 1〜3 件を提示する。

### 4. 止まった状態を立て直したい
Blocker を言語化すると、診断と最小の再始動ステップが返る。

### 5. 週次で進め方を見直したい
進捗、学び、進まなかった理由を review に残し、reroute する。

## 受け入れ条件
- Goal を入力し、保存できる
- Goal から milestone と quest の叩き台を生成し、保存できる
- Today 画面で quest の着手と完了ができる
- Blocker を登録すると、次の一手が記録される
- Weekly Review を保存できる
- 上記の主要操作に対応する event が残る
- Supabase 未設定でも fallback でローカル動作する

## v0.1 の設計原則
- Goal Setting を省略しない
- ユーザーに計画立案を求めすぎない
- 構造化 state を正本とし、event log を積み上げる
- 単一ユーザー、認証なし、Web アプリ最小構成で始める
- Quest 語彙は使うが、ゲーム感より前進感を優先する
