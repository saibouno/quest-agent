# Research Notes

Quest Agent の実装に効く外部レポートや調査メモをここで整理する。

## 置き方

- `docs/research/reports/`
  ローカルに置く PDF / docx の原本。Git では追跡しない。
- `docs/research/README.md`
  どの資料が何に効くかを短く残す索引。

## 現在の資料

### 目標を立てて実現するためのエビデンスベース・ワークフロー

- 使いどころ: Goal から Today の行動までをどう分解するか
- 効きそうな領域: Intake、Map、Today の導線、実行しやすい次の一手の設計

### 複数目標の同時運用と切替・停止の実務設計

- 使いどころ: 複数の目標をどう持ち替えるか、止める時に何を残すか
- 効きそうな領域: Portfolio、park / resume、同時進行数の扱い

### 回避・生産的先延ばし・自己認識・再開・撤退を統合した実務設計の深掘り報告書

- 使いどころ: ズレの検知、戻し方、撤退や保留の判断
- 効きそうな領域: Detect & Return、Mirror、Review、restart note の設計

## 運用メモ

- 原本はローカル保管を前提にする。
- 実装判断に残したい要点は、この README か `docs/` 配下の正式な資料へ要約して移す。
